Eat Ups

Eat Ups is a HTML5 game where the player’s goal is to do as many chin-ups as possible. It was built in 48 hours from February 9th to 11th for BC Game Jam 2018 in a team of five. Eat Ups was made with just HTML5 and JavaScript, which included some use of the JQuery JavaScript library. It is capable of being played on desktop and mobile. My teammates were Eric Cheng, Henry Ho, Peter Luk, and Steve Cho.

Gameplay

Overview

Eat Ups was created in response to the main theme of BC Game Jam 2018, which was “resolution”. The main avatar had to be someone who would be doing chin-ups to fulfill their New Year’s resolution of getting fitter. To add some challenge, we added an energy bar that the player must keep filled up. Each chin-up will reduce the amount of energy in the bar. The player can keep it filled up with the healthy food that passes by.

Contributions

I helped in the ideation of Eat Ups where we discussed simple games that we could complete within the 48 hours. I programmed the energy bar, user interface, game states, and all images except the food. I made use of the native shape drawing functions for the health bar and images within canvas. For the buttons and other images, I took advantage of creating HTML5 block elements and CSS’s layering functionality with the z-index. I also made the game responsive by resizing all HTML5 elements according to the display width, so it can be played on both desktop and mobile.

Challenge

One challenge I encountered was programming animations with frames created by Eric and Peter. I looked up solutions online that would simply cycle through the frames with JavaScript. I tried a few and ended up with one that worked, but it was not ideal. For some reason, the animation would randomly lag

Animation with JavaScript lagging

The animation we wanted to include would loop, so I suggested that we convert that into a GIF and I would position it with CSS accordingly. It ended up working out better than the other solution as it would not slow down.

Animation in GIF format

Reflection

In the end, Eat Ups was successful. We were able to implement all the necessary game elements we planned to implement. It was a game that players can play through and keep replaying.

If I could do this differently, I would want to make the desktop look better. Currently, the game takes up a 9:16 aspect ratio, which creates two large white bars on the sides. I would add a background image spanning the entire width of the window that changes according to what state the game is in.

I learned how to use JavaScript to create a simple HTML5 game using the canvas element that can be playable on both desktop and mobile. I also gained more experience with Git, which we used to manage all the files.
