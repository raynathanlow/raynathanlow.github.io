Sort Plus

Spotify companion app

Sort Plus sorts a user's saved albums by duration or release year.

It has the following features that makes it a progressive web app:

  • Work offline
  • Accessible via a link like a typical website
  • Installable like an app so that it can be launched from the device's home screen

Time Frame

November 2019 - February 2020

Languages and Tools

MERN (MongoDB, Express, React, Node.js), JavaScript, CSS, Mongoose, Git, Heroku, Figma

Motivation

I wanted to find an album saved in my Spotify library that I could listen to in its entirety within the limited amount of time I had.

Contributions

  • Research: I looked into what data was available from Spotify to sort my saved albums by. I also went through the Sonos and Spotify forums to see what similar features users requested.
  • Design: I designed the user experience, user interface, and icon.
  • Development: I wrote the back-end (database and server) code with MongoDB and Express. I wrote the front-end (user interface) code with React.
  • Deployment: I deployed Sort Plus to Heroku, a cloud platform that lets users put apps onto the Internet.

Challenge: Make it Work Offline

Sort Plus is designed to work offline which means that it must download the necessary data for it to work offline, which I had to program.

Control Which Data to Download

I can control which data is downloaded and saved on the device through code called service worker, but it was not configurable with the template I started out with.

Fortunately, I found a tutorial that taught me how me how to generate a new service worker to configure and use.

Change How Data is Requested and Sent

I wanted the service worker to only download data that is not already on the device, but there was a problem with how I initially wrote the code to manage how data is requested and sent.

Initially, one request sends all the user's saved albums, which forces them to redownload everything whenever their library has updated.

Now, one request sends only one saved album, so the service worker knows which data to download or not.

Implement Visual Feedback

Finally, I programmed the component that provides visual feedback as Sort Plus prepares for offline use and can start the download process, depending on its current state.

Reflection

Measuring Success

Sort Plus was successful because it functions the way I designed it to. Unfortunately, I don't find myself wanting to use it as much as I thought, but it may be due to a change in my listening habits.

What I Learned:

  • Minimum viable product (MVP): Developing a minimum viable product would have been helpful to test its usefulness before committing a lot of time into something I don't often use. Fortunately, the overall process helped me learn a lot.
  • MERN stack (MongoDB, Express, React, Node.js): Prior to this project, I only had experience with React and some backend (server and database) experience with PHP and MySQL. I learned how to write the backend with Express, which connects to MongoDB with Mongoose.
  • PWA (Progressive Web App): I learned more about progressive web apps so that I could make Sort Plus work offline. They can be accessible across different platforms and can be made to work offline for users who don't always have access to the Internet.
