Ancient Beast

Open source browser game

An open source turn based strategy browser game where players command an avatar who can summon creatures to defeat their opponents.

I contributed to the development of the game by resolving a couple of issues with its user interface and usability.

Contributions

  • Suggested: I suggested improvements to the resource bar design so that players better understand what actions they can take based on the available amount of resources.
  • Development: I modified the style of the in-game statistics with CSS to improve visual consistency. I implemented the resource bar improvements with JavaScript and CSS based on a maintainer's design. A maintainer is someone who manages what work needs to be done and what code will be included to the official version of the game.

Challenge: Fix and Improve Resource Bar

Players of Ancient Beast materialise and control creatures by casting abilities to defeat the opponent. I chose to resolve an issue where the resource bar sometimes renders past its bounds when the player hovers over it.

Resolved Issues and Suggested Improvements

I was able to resolve the issue by modifying its JavaScript code, but I noticed an opportunity to improve it during testing. I noticed that when the player interacts with an ability, the resource bar could show more information.

Originally, when the player hovers over an ability that requires more resources than the creature has, the resource bar changes colour and resizes to match the required amount of resources. It no longer shows the current amount of resources the creature has.

Fortunately, the maintainer I discussed with was able to redesign the resource bar that communicates more information to the player.

For example, a darker yellow indicates how much resources the ability would consume while yellow indicates how much resources would remain after the ability has casted.

Modified Preexisting Code to Implement Redesign

My first attempt at implementing the redesign was not very successful because I tried to change as little code as possible to avoid introducing more bugs. The redesign couldn't really be implemented because of how the resource bar was initially coded though.

After gaining a better understanding of the code, I was eventually able to implement the redesign.
